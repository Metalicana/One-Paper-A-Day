# One Paper A Day

This repository is a personal paper-reading log.

The scope is intentionally simple: for each paper I read, I upload a Markdown (`.md`) file that highlights the main ideas, problems, methods, and takeaways from that paper. In some cases, I may also keep the original PDF next to the note for reference.

## Purpose

The goal of this repo is to build a consistent habit of reading papers and capturing the most important ideas in a format that is easy to revisit later.

These notes are meant to be:

- Fast to write
- Easy to scan
- Focused on the parts of the paper that stood out to me

## Note Format

Each paper note may include sections like:

- Paper link
- Authors
- Problems the paper addresses
- Potential applications
- Method summary
- Personal observations or follow-up ideas

The exact structure does not need to be rigid. The main goal is to preserve the key insights from each paper in a readable Markdown file.

## How I Use This Repo

1. Read a paper.
2. Create a Markdown file for it in the relevant folder.
3. Write down the important highlights in my own words.
4. Optionally save the source PDF beside the note.

## Future Scope

As this repository grows, it may include:

- More topic folders
- A larger archive of paper notes
- More consistent templates for summaries
- Cross-links between related papers

## Summary

This is not a code-heavy repository. It is a lightweight knowledge base for research papers I have read, where each entry is primarily a Markdown summary of one paper.
